# Phase 04 Plan 01 — Observability Code Changes

## Objective
Implement stale-feed indicator (OBS-04), health badge honesty label (OBS-01), and
persistence error logging with re-raise (OBS-02).

## Tasks

### Task 1 — useAgentDecisions: stale feed tracking
- File: `src/hooks/useAgentDecisions.ts`
- Add `lastUpdated: Date | null` state, initialized to null
- Add `STALE_THRESHOLD_MS = 60_000` constant
- Compute `isStale` as `lastUpdated !== null && Date.now() - lastUpdated.getTime() > STALE_THRESHOLD_MS`
- Set `lastUpdated = new Date()` when initial fetch completes (in the `finally` block)
- Set `lastUpdated = new Date()` in the Realtime INSERT handler
- Export `lastUpdated` and `isStale` in return type

### Task 2 — AgentFeed: stale banner + health badge label
- File: `src/components/AgentFeed.tsx`
- Import `lastUpdated` and `isStale` from `useAgentDecisions` destructure
- Add amber stale banner above the content area when `isStale` is true
- In `AgentHealthCard`: add `title` attribute to the health percentage span indicating
  it is a latency-based estimate; append "(est.)" to the displayed percentage

### Task 3 — persistence.py: error logging + re-raise
- File: `trading_system/agents/persistence.py`
- In `save_decision` except block: change `logger.warning` → `logger.error`
- Add `raise` after the error log so callers see the failure
- Update the module docstring design contract note to reflect the new behavior

## Success Criteria
- TypeScript: no new type errors introduced
- `useAgentDecisions` exports `lastUpdated` and `isStale`
- `AgentFeed` conditionally shows stale banner
- `AgentHealthCard` labels health percentage as estimate
- `persistence.py` raises on save failure (not just logs)
